1) What is Selenium WebDriver?
It is an API that allows your code to directly control a browser.
Your code sends commands like:

open page
click element
type text
get value

The browser executes those commands.
```
WebDriver driver = new ChromeDriver();
driver.get("https://example.com");
```

2️⃣ Browser Drivers

Selenium cannot control browsers directly.
It uses browser-specific drivers.

Examples:

ChromeDriver → for Google Chrome

GeckoDriver → for Mozilla Firefox

Edge WebDriver → for Microsoft Edge

3️⃣ Selenium Client Libraries

These allow you to write Selenium code in different languages.

4️⃣ Web Elements

Selenium interacts with HTML elements on a webpage.
Examples:

buttons
text fields
links
dropdowns

Example:
driver.findElement(By.id("username")).sendKeys("admin");

This means:
Find element with id = username
Type "admin"

5. Simple Selenium Example

WebDriver driver = new ChromeDriver();

      .sendKeys("Selenium WebDriver");

1️⃣ Mouse Hover
 (Move to Element)

Used when a menu appears only after hovering.

Example scenario:

Menu
 └── Submenu appears when mouse hovers

Code:

import org.openqa.selenium.interactions.Actions;
WebElement menu = driver.findElement(By.id("menu"));
Actions actions = new Actions(driver);
actions.moveToElement(menu).perform();

✔ Moves the mouse over the element.

2️⃣ Right Click
 (Context Click)

Simulates a right-click.

WebElement element = driver.findElement(By.id("button"));
Actions actions = new Actions(driver);
actions.contextClick(element).perform();

✔ Opens context menu.

3️⃣ Double Click

Simulates double-click action.

WebElement element = driver.findElement(By.id("doubleClickBtn"));
Actions actions = new Actions(driver);
actions.doubleClick(element).perform();

✔ Useful for UI elements that trigger double-click events.

4️⃣ Drag and Drop

Used when dragging an element and dropping it somewhere.

Example:

Drag item → Drop into container

Code:

WebElement source = driver.findElement(By.id("drag"));
WebElement target = driver.findElement(By.id("drop"));
Actions actions = new Actions(driver);
actions.dragAndDrop(source, target).perform();

✔ Moves element from source → destination.

5️⃣ Keyboard Events

Keyboard actions simulate shortcut keys.

Example: Press ENTER

driver.findElement(By.id("search"))
      .sendKeys(Keys.ENTER);

Example: Copy-Paste

WebElement field = driver.findElement(By.id("input"));
field.sendKeys("Selenium");
Actions actions = new Actions(driver);
actions.keyDown(Keys.CONTROL)
       .sendKeys("a")
       .sendKeys("c")
       .keyUp(Keys.CONTROL)
       .perform();

Common keyboard keys come from: :---[ org.openqa.selenium.Keys ]

Examples:

Key			Usage
ENTER		submit forms
TAB			move to next field
ESCAPE		close popups
CONTROL	shortcuts

Sysnchronisation

AJAX requests
JavaScript rendering
Network delay
Dynamic DOM updates

If Selenium tries to interact before the element is ready, tests fail.

Types of Waits in Selenium
Wait Type	Purpose
Implicit Wait	Global wait for elements
Explicit Wait	Wait for a specific condition
Fluent Wait	Advanced explicit wait with polling

-----------------------------------------------------------------

1️⃣ Implicit Wait

Implicit wait tells Selenium:
"Wait up to X seconds for an element before throwing an exception."

Example:
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

✔ Applies to all elements globally.

Limitations:
Cannot wait for specific conditions
Less control

2️⃣ Explicit Wait

Explicit wait waits for a specific condition.

Classes used:

WebDriverWait
ExpectedConditions

```
import org.openqa.selenium.support.ui.WebDriverWait;
import org.openqa.selenium.support.ui.ExpectedConditions;

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));

WebElement element = wait.until(
        ExpectedConditions.visibilityOfElementLocated(By.id("loginBtn"))
);

element.click();

```
Wait up to 10 seconds until element becomes visible

Common conditions:

Condition			Purpose
visibilityOfElementLocated	element visible
elementToBeClickable		element clickable
presenceOfElementLocated	element present in DOM
textToBePresentInElement	verify text appears

Example:

wait.until(ExpectedConditions.elementToBeClickable(By.id("submit"))).click();

✔ Most commonly used wait in automation frameworks.

3️⃣ Fluent Wait

Fluent wait is an advanced version of explicit wait.

Features:
Custom timeout
Custom polling interval
Ignore specific exceptions

Example:
```
import org.openqa.selenium.support.ui.FluentWait;

Wait<WebDriver> wait = new FluentWait<>(driver)
        .withTimeout(Duration.ofSeconds(20))
        .pollingEvery(Duration.ofSeconds(2))
        .ignoring(NoSuchElementException.class);

WebElement element = wait.until(
        driver -> driver.findElement(By.id("loginBtn"))
);

element.click();
```
